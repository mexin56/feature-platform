# 特征调度管理平台 实施路线图

> Spec:`docs/superpowers/specs/2026-06-12-feature-platform-design.md`
> 每阶段产出可独立运行、可测试的软件;阶段计划在前一阶段完成后编写,避免脱离实际。
> 环境:conda scpy310;命令均在 `D:\feature-platform` 下执行。

## Phase 1a — 项目骨架与后端核心(本期,详见 2026-06-12-phase1a-backend-core.md)

config/db/域模型(用户/项目/成员/审计)、secrets(Fernet 密钥)、认证(bcrypt+JWT)、
用户管理、项目隔离、操作留痕。**产出:可登录、建项目、权限隔离的最小后端。**

## Phase 1b — 调度域模型与调度内核

- 连接管理(MySQL / Spark ThriftServer,密码 Fernet 加密,测试连通)
- Workflow CRUD:DAG(节点+边 JSON)校验(无环/依赖存在)、版本化、上线/下线
- WorkflowRun / TaskInstance 模型与状态机
- tick 调度器(5s):Cron 水位(croniter)→ 建实例;依赖推进;孤儿清理
- 执行器:ProcessPoolExecutor、`UPDATE…WHERE state='queued'` 原子抢占、心跳、
  超时强杀、重试(up_for_retry)、失败策略(继续/结束)
- 模板变量渲染:`{{ ds }}` / `{{ data_interval_start }}` / `{{ data_interval_end }}`
- 插件:`duckdb_sql`(产 Parquet)、`python_script`(托管脚本子进程)
- API:手工触发、重跑、失败点续跑、置成功、终止、补数(串行/并行)
- **产出:能定时调度 DAG、本地插件端到端可跑的最小调度系统。**

## Phase 2 — 特征域与在线特征服务

- FeatureGroup(版本/主键/事件时间/TTL/online 开关/离线落地)+ Feature(口径)+ 血缘边表
- 生产即注册:任务成功回写特征组"最近产出时间/行数"
- 插件:`sql_pushdown`(Spark/MySQL 下推,行数校验)、`dependent`(跨工作流依赖)、
  `materialize`(水位增量 → online_store.db 幂等 upsert)
- 在线查询:`POST /api/online-features`(API Key 认证、TTL 过期标记、调用量统计)
- **产出:特征全生命周期闭环——注册→调度生产→物化→在线查询。**

## Phase 3 — 监控与告警

- 大盘统计 API(今日成功/失败/运行中、耗时 Top、失败列表)
- 特征质量:行数/主键去重数/空值率落库,环比突变告警(默认降幅>50% 可配)
- 物化滞后告警、SLA(应在 HH:MM 前完成)
- Webhook 渠道(飞书卡片预置)+ 站内告警中心
- **产出:失败/质量/SLA 三类告警可达,可观测。**

## Phase 4 — 前端与端到端

- React 18 + Vite + AntD5 + ECharts + React Flow 骨架(:5174 dev,构建产物由 :8100 托管)
- 9 页面:登录/工作台/特征组(血缘图)/工作流编辑器(拖拽 DAG+SQL 编辑器)/
  调度与补数/实例监控(DAG 着色+日志)/在线特征调试台/告警中心/系统管理
- 端到端冒烟:建特征组→编排→触发→物化→在线查询
- **产出:浏览器全功能可用,验收交付。**
